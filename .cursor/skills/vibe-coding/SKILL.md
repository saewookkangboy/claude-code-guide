# 바이브 코딩 스킬

바이브 코딩은 **역할 + 상황/목표 + 제약 + 출력 형태**로 요청을 구조화해 AI 응답 품질을 높이는 방식이다.

## 언제 사용하는가

- 사용자가 기능 구현, 기획, UI, 리팩터링, 테스트, 문서화를 요청할 때
- 요청이 한 줄이거나 맥락이 부족할 때 → 네 가지 요소로 나눠 확인하거나 제안
- 사용자가 "바이브 코딩으로", "템플릿으로 정리해줘"라고 할 때

## Part와 출력 가이드

| Part | 목적 | 출력 예시 |
|------|------|-----------|
| 기획·설계 | API/화면/플로우/DB 정리 | 목록, 플로우, 트레이드오프 |
| 기능 구현 | 코드 작성 | 코드 + 필요 시 짧은 설명 |
| UI·스타일 | 감각/스타일 반영 | 컴포넌트 + 스타일 |
| 리팩터링 | 구조 개선, 동작 유지 | 변경 코드 + 요약 |
| 테스트·디버깅 | 검증/원인 분석 | 테스트 코드 또는 원인+수정안 |
| 문서화 | README/API/주석 | 문서 초안/스니펫 |

## 동작 원칙

1. **요청이 구조화되어 있으면** 역할/상황/제약/출력을 그대로 따르고, 요청한 출력 형태만 낸다.
2. **요청이 짧거나 모호하면** "역할 / 상황 / 제약 / 출력" 중 빠진 것을 질문하거나, 프로젝트 루트의 `prompts/` 템플릿(part1~part6, template-blank.md)을 참고하라고 안내한다.
3. **한 번에 한 Part만** 다룬다. 여러 Part가 섞여 있으면 우선순위를 정해 하나씩 처리한다.

## 프로젝트 내 리소스

- **템플릿**: `prompts/part1-planning.md` ~ `prompts/part6-documentation.md`, `prompts/template-blank.md` — 복사용 예시 및 빈 칸 템플릿

이 스킬은 프로젝트에 포함되어 있으므로, 이 저장소를 클론한 뒤 Cursor에서 바로 바이브 코딩 구조로 요청하면 된다.
