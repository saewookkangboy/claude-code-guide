# Part 6: 문서화 — 복사용 프롬프트

아래 블록을 복사한 뒤 `[ ]` 안과 항목만 프로젝트에 맞게 수정하세요.

---

## 예시 1: README 로컬 실행 방법

```
[문서화] 이 프로젝트의 로컬 실행 방법을 README에 추가해줘.
- 역할: 문서 작성. 새로 합류하는 개발자가 따라할 수 있게.
- 상황: Node 20 + pnpm, 환경 변수 .env.example 필요, DB는 Docker Compose로 띄움.
- 제약: 기존 README 구조 유지. "개발 환경 설정" 섹션 하나만 추가. 명령어는 복사해서 쓸 수 있게.
- 출력: 추가할 Markdown 블록만. 코드 블록 포함.
```

---

## 예시 2: OpenAPI 스펙 요약

```
[문서화] 방금 만든 주문 API 3개에 대한 OpenAPI 스펙 요약을 만들어줘.
- 역할: API 문서화.
- 상황: POST /orders, GET /orders/{id}, PATCH /orders/{id}/cancel. 요청/응답 필드와 상태 코드만 정리.
- 제약: OpenAPI 3.0 형식. 인증은 Bearer 토큰이라고만 명시. 예시 값 하나씩 포함.
- 출력: paths 와 schemas 만 있는 YAML 스니펫.
```

---

## 빈 칸 템플릿

```
[문서화] ________(대상: README/API/주석 등) ________(작성/추가) 해줘.
- 역할: 문서 작성. ________(대상 독자: 새 개발자/API 사용자 등).
- 상황: ________(대상, 필요한 내용).
- 제약: ________(포맷: Markdown/OpenAPI, 언어, 분량).
- 출력: ________(초안 블록만 / YAML 스니펫 등).
```
